1. Bienvenida

Tubelight es un overlay CRT que se planta encima de cualquier aplicación de Windows y procesa lo que muestre — un juego retro, un emulador, una terminal, un reproductor de vídeo, lo que sea — para que se vea como en un monitor de tubo de verdad: con líneas de escaneo, máscara de fósforo, halo del cristal, persistencia del rojo y barrido sutil del haz.

No es un filtro de captura ni un shader pegado a un emulador. Es una ventana real de Windows que captura el escritorio por debajo de ella con DXGI Desktop Duplication, aplica una pipeline de 8 pasadas OpenGL en tiempo real y muestra el resultado a 60 FPS encima. Cualquier app que se ejecute debajo — DirectX, OpenGL, Vulkan, software puro — sale renderizada por el tubo.

La palabra que define el diseño es fidelidad: cada uno de los 16 perfiles CRT está calibrado contra un manual de servicio real del fabricante, con cromaticidades de fósforo citadas, dot pitch verificado y persistencia por canal. No es "un CRT genérico con sliders": es Sony PVM-8220 vs. Sony BVM-20F1U vs. Sony FW900 vs. Sharp X68000 CZ-602D, cada uno con su look específico.

Qué es Tubelight, en una página

Lo que Tubelight hace: pintar encima de tu pantalla un cristal CRT realista, calibrado contra hardware histórico. Cuatro modos de uso, 16 monitores de tubo bundle, 7 perfiles de señal (RF, composite NTSC/PAL, S-Video, SCART RGB, Component, RGB VGA), pipeline OpenGL 8-pass, captura PNG y vídeo MP4 vía ffmpeg, audio CRT XAudio2.

Lo que NO hace: emular hardware. Tubelight no sabe nada de juegos, ROMs, emuladores ni señal de vídeo real — sólo coge la imagen ya renderizada por otra app y la transforma. No es un substituto de RetroArch / Mednafen / openMSX, sino un compañero que añade el monitor que esas apps no traen.

Por qué importa la diferencia: muchos "filtros CRT" suman scanlines como una capa con multiply. Tubelight modela cromaticidad de fósforo, máscara, persistencia por canal, distorsión geométrica y halación por separado, y cada perfil tiene los valores reales del monitor que imita. La diferencia entre un PVM-8220 (aperture grille, P22 estándar) y un BVM-20F1U (aperture grille Super Fine Pitch, P22 con calibración broadcast) es visible — porque viene de los manuales de servicio.

Casos de uso

Emuladores standalone: Mednafen, MAME, BlastEm, openMSX, FCEUX. Lánzalos, lánza Tubelight encima en modo ventana o target, elige perfil → la salida del emulador pasa por el tubo aunque el emulador no tenga un solo shader.

Emuladores libretro / RetroArch: para integrar dentro del runner, exporta el perfil actual como preset .slangp con tubelight --export-slangp out.slangp --profile X --signal Y y cárgalo en RetroArch. O sólo lánza el overlay encima y olvídate del flujo libretro.

Juegos retro modernos: UFO 50, Sea of Stars, Shovel Knight, Children of Morta, Crypt of the NecroDancer. Pixel-art rendido en panel plano nunca da el feel del original; pásalo por un pvm-8220 o un commodore-1084s y los pixeles "vibran" como tocaba.

Reproductores de vídeo: mpv, VLC, navegador, YouTube. Útil para material de archivo (VHS rips, demos, MV ochenteros). El perfil tv-bw-p4 con signal rf convierte cualquier clip moderno en un televisor de tubo del 75.

Terminales y código: cmd, PowerShell, WSL, VS Code. terminal-p31 (verde Apple //e) o terminal-p3-amber (ámbar IBM 5151) sobre tu editor le da ergonomía vintage real, sin abandonar las herramientas modernas.

Vídeo de demostración / streaming: con el modo recordable (Ctrl+Alt+R), apps como Snipping Tool, Game Bar y OBS pueden grabar el overlay procesado (no la fuente cruda). Útil para mostrar tu setup como aparece en pantalla.

No-objetivos (qué Tubelight NO hace, a propósito)

  • No es un emulador. Si quieres jugar a SMW necesitas Snes9x; Tubelight pinta encima.
  • No genera señal. No hay generador de patrones internos, ni sincronía 31 kHz / 15 kHz. Si la app de abajo está a 144 Hz, Tubelight la procesa a 144 Hz (no la baja a 60).
  • No quita lag. El pipeline añade ~1 frame por DXGI + ~1 frame por la propia composición. Si te importa la latencia para juegos competitivos, el modo low_latency lo baja al mínimo (vsync off + cap 240 FPS) pero sigue siendo un overlay, no un wrapper.
  • No graba la app subyacente con audio sincronizado. Ctrl+Alt+V graba el overlay procesado a MP4 H.264 (vídeo). Para audio del juego usa OBS con Window Capture WGC (compatible con modo recordable).
  • No emula consolas, mappers, audio de chips ni cualquier otra cosa que tu emulador haga. Sólo el monitor.
  • No persiste configuración entre cuentas de usuario. Cada usuario tiene su %APPDATA%\Tubelight\settings.json.
  • No es portable. Requiere Windows 10 build 2004+ / 11 para DXGI Desktop Duplication moderna + DWM composite. Linux planeado para v1.1 (sin recordable; usa OBS-Wayland o gpu-screen-recorder).

2. Instalación

Tubelight v0.1.3 (Windows; Linux próximamente en v1.1) se distribuye como un único tubelight.exe (~1.2 MB) más DLLs de runtime (epoxy-0.dll, glfw3.dll) y la carpeta profiles/ con los 16 CRT + 7 signal. No requiere instalador; copia la carpeta donde quieras y lanza el ejecutable.

Doble clic en tubelight.exe lanza el overlay en modo windowed con el último perfil usado (o pvm-8220 / composite_ntsc la primera vez). El binario abre una ventana de Win32 redimensionable que muestra el área de escritorio que tiene debajo, procesada por la pipeline.

Requisitos

  • SO: Windows 10 build 2004 (May 2020) o superior; Windows 11 recomendado.
  • GPU: cualquiera compatible con OpenGL 4.5 core profile (Intel HD Graphics 4400 / 2014, NVIDIA Kepler / 2012, AMD GCN 1.0 / 2012 en adelante). Una iGPU moderna sobra.
  • CPU: un núcleo libre. La pipeline corre en GPU; la CPU sólo presenta frames y mueve datos por DXGI.
  • RAM: 200 MB de uso típico.
  • Disco: 5 MB para el ejecutable + DLLs + profiles. Capturas PNG ~2 MB cada una a 1920×1200, vídeo MP4 ~30-60 MB/min a 60 FPS.
  • ffmpeg (opcional): para grabación MP4 con Ctrl+Alt+V. Debe estar en PATH. Si no está, el resto funciona — la grabación queda deshabilitada con un toast claro.
  • Magnification API: incluida en Windows desde XP. Sólo se usa en modo recordable; no requiere instalación.

Descarga e instalación

Opción A — zip de release (recomendada): descarga el tubelight-<versión>-win64.zip más reciente de la página de releases, descomprime en C:\Tools\Tubelight\ (o donde te apetezca). Doble clic en tubelight.exe.

Opción B — build local desde fuentes: necesitas Visual Studio 2022 BuildTools (con C++), vcpkg, CMake 3.26+. Ejecuta scripts\build_windows.bat desde la raíz del repo. El build produce build\windows-vcpkg\Release\tubelight.exe y copia DLLs + profiles al lado. Tarda ~3-5 min en frío, ~1 min incremental.

La primera ejecución crea %APPDATA%\Tubelight\settings.json con defaults. Las capturas van a %USERPROFILE%\Pictures\Tubelight\ por defecto; puedes cambiarlo en el menú Capture → Browse.

Build local Windows
> scripts\build_windows.bat
Verificar versión
> tubelight --version

Primera ejecución

Doble clic en tubelight.exe. Aparece una ventana 1280×960 movible. Lo que se ve dentro es la zona de escritorio que la ventana tiene debajo, ya procesada por la pipeline (perfil pvm-8220 + signal composite_ntsc por defecto).

Mueve la ventana sobre lo que quieras procesar — un emulador, un vídeo, una terminal. Arrastra los bordes para redimensionar; la pipeline reconfigura los FBOs automáticamente.

Ctrl+Alt+M abre el menú in-app (cinco pestañas: Profile / Image / Capture / Audio / Help). En Profile elige el monitor CRT y la señal. En Image ajusta scanlines, mask, bloom, persistence. En Capture configura la carpeta de capturas. En Audio el flyback whine. En Help atajos y autoría.

Ctrl+Alt+Q cierra Tubelight.

Desinstalar / portabilidad

Tubelight no toca el registro de Windows. Para desinstalar:

  1. Borra la carpeta donde está tubelight.exe.
  2. Borra %APPDATA%\Tubelight\ (settings + presets de usuario).
  3. Opcional: borra %USERPROFILE%\Pictures\Tubelight\ (capturas y vídeos).

Para backup: zip de la carpeta del binario + %APPDATA%\Tubelight\. Restaurar es descomprimir donde estaban.

3. Tour de la interfaz

Tubelight tiene cuatro elementos visuales con los que el usuario interactúa: la ventana overlay, el menú in-app (cinco pestañas), el HUD de estado (esquina superior derecha, opcional) y los toasts de notificación (esquina inferior izquierda, efímeros).

INFO: El menú está hecho con Dear ImGui. Si se hace muy grande para tu pantalla, arrastra su título para moverlo. Sus posiciones no persisten todavía (planeado para v1.1).

Ventana overlay

Por defecto, ventana Win32 redimensionable con marco, 1280×960. Lo que muestra es el contenido del escritorio que tiene debajo, ya procesado.

  • Arrastra el título para mover.
  • Arrastra cualquier borde para redimensionar. Durante el drag, la pipeline sigue procesando (sin congelar a frame final como hacen muchas apps GLFW).
  • Doble clic en el título: maximizar / restaurar.
  • Ctrl+Alt+Enter: alterna fullscreen runtime preservando el aspect ratio elegido (sin tocar GWL_STYLE, ver §13 sobre el bug evitado).
  • Ctrl+Alt+C: alterna click-through. Cuando está activo, los clicks atraviesan la ventana hacia la app de debajo. Útil cuando quieres jugar / editar / leer sin que Tubelight robe input. Stricto per-session: no persiste entre lanzamientos.

Menú in-app (5 tabs)

Ctrl+Alt+M lo abre o cierra. Estética dark con paleta de 6 acentos por categoría (sky / teal / amber / coral / lavender / mint) y tooltips parchment cream+gold que explican cada widget en castellano.

Tab Profile (sky): dropdown CRT (16) + dropdown Signal (7) + slider Global intensity 0..2.0× sobre el baseline del perfil. Al cambiar de perfil, los demás sliders del menú Image saltan a los valores baseline del nuevo perfil.

Tab Image (teal): grupos Scanlines / Beam, Phosphor mask, Phosphor colour (mono), Bloom / Halation, Persistence, Composition (barrel, vignette, gamma display, aspect ratio, bezel), Pass toggles (8 checkboxes). Estructura distinta para CRTs color vs. monocromos (ver §8).

Tab Capture (amber): carpeta de capturas con campo + botón Browse (file picker nativo IFileOpenDialog) + Apply + Default. Combo Record source (overlay / full monitor / custom rect).

Tab Audio (coral): volumen master + toggle flyback whine + toggle degauss thump al cambio de perfil.

Tab Help (lavender + mint): tabla de atajos, créditos, link a este manual, versión + git SHA.

HUD de estado y toasts

HUD (Ctrl+Alt+H): esquina superior derecha. Muestra perfil actual / signal / modo (windowed / fullscreen / target / region) en mono pequeño con fondo translúcido. Persiste hasta cerrarlo otra vez.

Toasts: notificaciones efímeras (3 s) en esquina inferior izquierda. Aparecen al guardar screenshot, iniciar/parar grabación, cambiar profile, error de ffmpeg, activación de recordable mode. No bloquean input.

REC dot: punto rojo parpadeando arriba a la izquierda mientras está grabando vídeo. Independiente de toasts.

4. Tu primer overlay CRT

Un recorrido de 60 segundos para ver el efecto en tu propio escritorio.

  1. Abre cualquier cosa pixel-art o de texto: un emulador con un juego cargado, una terminal con texto verde, una imagen de captura en Windows Photos, lo que tengas a mano. Tiene que ocupar suficiente espacio como para que un overlay 1280×960 pueda taparlo.
  1. Doble clic en tubelight.exe. Aparece la ventana overlay. Probablemente esté en el centro del monitor primario; arrástrala hasta colocarla encima de tu contenido.
  1. Pulsa Ctrl+Alt+M para abrir el menú. En el tab Profile (icono sky azul), abre el dropdown CRT. Empieza por algo familiar:
  2. - pvm-8220 si quieres un monitor profesional Sony de los 80–90 muy común.
  3. - commodore-1084s si quieres TV/monitor consumer slot-mask con ese feel de Amiga y home computer 80.
  4. - terminal-p31 si lo que tienes debajo es texto y quieres verde 70s.
  5. - mac-classic-white si quieres 1-bit blanco-azulado puro (sí, posterize a 2 niveles, sí, se ve igual que un Mac SE en 1989).
  1. En el mismo tab, abre el dropdown Signal. Para juegos retro consola → composite_ntsc (lo que tu Mega Drive sacaba por RCA amarillo). Para terminal → rgb_vga (clean, pixel-perfect). Para sentir el ruido de TV antigua → rf.
  1. Mira lo de debajo. Las scanlines son visibles, los píxeles ya no son ladrillos planos, el blanco tiene halo, los rojos tienen estela mínima cuando algo se mueve.
  1. Mueve el slider Global intensity en el mismo tab Profile. A 0 se desactiva todo el efecto (passthrough total). A 1 es el baseline calibrado del perfil. A 2 está exagerado para fotografía o para gente que quiere SCANLINES BIEN GORDAS.

Ya está. El resto del manual son los detalles.

INFO: Si arrastras la ventana muy rápido, durante el drag verás el pipeline procesando a tirones (DXGI no actualiza durante mouse-capture). Suelta el botón y vuelve a 60 FPS.

5. Modos de overlay

Cuatro modos, todos disponibles desde la CLI o desde el menú una vez arrancado. Eliges según el caso.

Windowed (default)

Ventana Win32 redimensionable con marco. Es el modo por defecto cuando haces doble clic. Útil para tener Tubelight visible mientras trabajas en otra cosa al lado. Soporta drag, resize, click-through, snap-to-aspect y todos los atajos.

CLI: tubelight --overlay [--size W H] [--profile X --signal Y]

> tubelight --overlay --profile pvm-8220 --signal composite_ntsc --size 1280 960

Fullscreen runtime (Ctrl+Alt+Enter, IF-safe)

Sin marco, monitor entero, encima de todo. Preserva el aspect ratio elegido en el tab Image (4:3, 5:4, 16:10, 16:9, 21:9, Fill).

Implementación: WM_NCCALCSIZE override que colapsa título y bordes a 0 px sin tocar GWL_STYLE, más SetWindowPos con monitor.height − 1 para mantener la línea transparente de 1 px abajo que bloquea Independent Flip de Win11 y deja la ventana en el path compositado DWM (necesario para que WDA_EXCLUDEFROMCAPTURE se respete). Ver §13 Solución de problemas y las lecciones aprendidas en docs/manual/ por qué NO se hace de la forma "obvia".

Desde CLI: tubelight --overlay-fullscreen [--profile X --signal Y] arranca directamente sin pasar por windowed.

Desde overlay: Ctrl+Alt+Enter alterna en runtime preservando aspect.

> tubelight --overlay-fullscreen --profile pvm-8220 --signal composite_ntsc

Target window (Ctrl+Alt+T)

Ata Tubelight a una ventana concreta. La ventana se mueve, Tubelight la sigue cada frame. La ventana se redimensiona, Tubelight se redimensiona. La ventana se minimiza, Tubelight se esconde.

Pensado para emuladores standalone (Mednafen, openMSX, MAME, FCEUX). Lánzalos sin shaders, lanza Tubelight encima en target mode → ahora son monitores PVM o lo que elijas.

Click-through automático: el target sigue recibiendo input. No interfiere con teclado ni ratón del juego.

Dos formas de target: por título substring o por PID. Por PID es más robusto (no confunde con otras ventanas).

> tubelight --overlay-target "Mednafen" --profile pvm-8220 --signal composite_ntsc

> tubelight --overlay-target-pid 12345 --profile sharp-x68k-cz602d --signal rgb_vga

Region fija (--overlay-region)

Pinta sobre un rectángulo fijo del monitor sin seguir ninguna ventana. Útil para tener un área CRT estática (ej: "esta parte de mi setup siempre es Sega Mega Drive con composite NTSC") aunque las apps debajo cambien.

Click-through siempre activo. No toma input.

Coordenadas: x,y = esquina superior izquierda; w,h = ancho y alto. En píxeles del monitor 0 por defecto.

> tubelight --overlay-region 200,150,1280,960 --profile commodore-1084s --signal composite_pal

6. Perfiles de CRT — catálogo de los 16 monitores

Tubelight bundle 16 perfiles. Cada uno trae mask type, dot pitch, diagonal, aspect, frecuencias de barrido, cromaticidad de fósforo y persistencia por canal — todo verificado contra manual de servicio o fuente equivalente (crtdatabase, ManualsLib, archive.org, gamesx.com, Wikipedia EIA phosphor table). Las URLs concretas están en el JSON de cada perfil (profiles/crts/<id>.json → campo source.url).

La galería siguiente es la misma testcard de referencia procesada por cada perfil con signal composite_ntsc. Cambia el signal en §7 si quieres ver cómo varía la cadena dentro del mismo monitor.

6.1 Sony PVM-8220 (Trinitron 8", 1985)

Profesional broadcast pequeño, aperture grille (Trinitron), dot pitch 0.30 mm, 8" diagonal, 4:3 nativo. Fósforo P22 estándar, primarios SMPTE-C. Persistencia por canal asimétrica (R lento, GB rápido → trail rojo).

Look: brillante, verticales nítidas (Trinitron), scanlines marcadas (240 visibles en NTSC), dos hilos de damping visibles a contraluz que cruzan el frame horizontalmente. Es el monitor más usado en testing de juegos retro NTSC por su precisión cromática y dot pitch fino.

7. Perfiles de señal — los 7 paths de la cadena

Cómo llegaba el vídeo al monitor en su época. Cada signal modela un conjunto distinto de artefactos físicos: ruido aditivo, chroma smear, dot crawl, ringing, ghosting, banding rainbow. Mismo monitor + signal distinto = imagen radicalmente diferente.

La galería usa el PVM-8220 fijo para que la única variable sea la señal.

7.1 RF (broadcast / antena)

Lo peor de lo peor en calidad — y lo más nostálgico para muchos. Modulación analógica a banda de TV (canal 3/4 o coaxial). En Tubelight: ruido aditivo alto, dot crawl muy visible, ringing horizontal pronunciado, ghost leve. Es lo que veías al conectar la NES por adaptador RF a una TV vieja.

7.2 Composite NTSC (consumer RCA, NTSC 525-line)

RCA amarillo. Una sola línea para luma + chroma multiplexados. Modelos clave en Tubelight: chroma smear (los bordes de color se difuminan horizontalmente porque chroma va sub-sampleada), dot crawl (patrón estriado en bordes saturados — el batido entre frecuencia portadora y luma), ringing moderado.

Es lo que sacaban Mega Drive, SNES, NES, PlayStation, N64 por defecto. Para juegos NTSC de consola es el signal que tu hardware veía.

7.3 Composite PAL (consumer, 625-line, EBU)

Equivalente europeo. Línea de retardo de chroma (Phase Alternating Line) cancela las inversiones de fase entre líneas adyacentes → menos rainbow banding que NTSC, más estabilidad de color. 625-line, 50 Hz. Tubelight modela menos dot crawl, chroma smear similar, scanlines a 288 visibles.

7.4 S-Video (Y/C, luma + chroma separados)

Dos pins: uno para luma (Y), otro para chroma (C). Sin multiplexado = sin dot crawl, sin chroma smear horizontal (sigue habiendo subsampling vertical leve). Mismo ruido base que composite. Es la mejor mejora calidad/coste vs composite — muchos hardcore retro pillaban cables S-Video para SNES (s-video by default) y Saturn.

7.5 SCART RGB (RGBS, casi clean)

RGB analógico + sync separado, viaja por cable SCART (estándar europeo). Sin multiplexado, sin subsampling de chroma. Sólo queda el ruido analógico de cable + el shadow/aperture-grille mask del monitor. Es la salida "clean" del retrogaming europeo: BVM/PVM + cable SCART RGB era el setup definitivo de los 90.

7.6 Component (YPbPr, separación diferencial)

Y + Pb + Pr en tres cables RCA. Separación luma/chroma diferencial → calidad equivalente a SCART RGB con cables más prácticos. Usado en consumer high-end EE. UU. (PlayStation 2 component, Xbox component, ciertos DVD).

7.7 RGB VGA (passthrough limpio, pixel-perfect)

Sin artefactos analógicos significativos. Es lo que veías en un monitor PC del 90+ con cable VGA estándar. Tubelight aplica esto como passthrough digital: cero ruido, cero smear, cero ringing. Las únicas distorsiones que verás vienen del perfil CRT (mask, scanlines, fósforo, bloom) — útil para aislar la contribución del monitor sin que la señal añada nada.

8. Ajustes finos — anatomía del menú Image

El tab Image es donde se vive el 80% del tiempo después de elegir perfil. Cada slider modela un fenómeno físico del CRT. Lo que sigue va sub-sección por sub-sección con la física que hay detrás, los rangos prácticos y cuándo conviene cada uno.

8.1 Scanlines / Beam (4 sliders)

Scanline strength (0..1): cuánto se oscurece el espacio entre líneas. 0 = sin scanlines (panel plano). 1 = máximo. Realista en CRTs de consumer: 0.6–0.8. En monitores broadcast finos (BVM): 0.3–0.5 (porque la línea está mejor enfocada y casi tapa el espacio).

Beam width (0.5..2.0): ancho del haz. <1 = haz más fino → scanlines más visibles, imagen más "crisp". >1 = haz más gordo → líneas se solapan, imagen más blanda.

Gamma (1.8..2.6): gamma del cañón CRT (no del display). Físicamente: la curva voltaje→luminancia del propio tubo, distinta de la corrección de salida sRGB. Valor estándar EBU/SMPTE: 2.2. PVMs broadcast suelen calibrarse a 2.5 (la curva original NTSC). Valores altos = sombras más profundas, negro más negro.

Scanline count: número de líneas visibles. 240 (NTSC), 288 (PAL), 480 (VGA). Determina cuántas scanlines pinta el shader en tu resolución de ventana — independiente del aspect ratio.

La trampa común: scanline count != resolución de la fuente. Si tienes una imagen 320×240 y eliges 480 scanlines, ves 480 líneas pero cada par de líneas comparte el mismo pixel fuente. Apparent fix: usar el conteo NTSC/PAL real (240/288) en juegos retro de consola.

8.2 Phosphor mask (color CRTs)

Sólo para CRTs color (queda greyed-out en mono).

Type combo: 6 tipos de máscara modelados. - shadow_mask: rejilla de puntos R-G-B. NEC MultiSync, mayoría de TVs consumer. - aperture_grille: rejilla vertical (Trinitron). Verticales nítidas. - slot_mask: ranuras verticales (Commodore 1084S). - diamond_mix: variante slot con triadas en diamond (algunos Hitachi). - none: sin máscara (debug). - auto: usa el del perfil cargado.

Strength (0..1): cuán visible es la máscara. 0 = invisible (interpolada). 1 = máscara dura. Valores realistas: 0.4–0.7. En monitores fine-pitch (0.28 mm): 0.2–0.4 (porque la máscara es físicamente más pequeña y se ve menos).

Pitch (0.20..0.65 mm): tamaño de la máscara. Cambia el espaciado del patrón.

8.3 Phosphor colour (monocromos)

Sólo para CRTs mono (queda greyed-out en color).

Tint R/G/B (0..2.0): cromaticidad del fósforo. Define qué color domina. Para P31 verde: (0.10, 1.30, 0.20). Para P3 ámbar: (1.40, 0.65, 0.05). Para P4 blanco: (0.95, 1.00, 1.10).

Presets: P31 verde, P3 ámbar, P4 blanco, P1 oscilo, custom. El custom guarda los valores en el preset al exportar.

Posterize levels (0..16): número de niveles tonales tras aplicar el fósforo. 0 = analógico (sin posterizar, gradiente continuo). 2 = 1-bit (sólo dos niveles, Mac Classic). 6 = típico terminal pre-greyscale. 16 = 4-bit cuasi-continuo. Cambia el feel radicalmente.

8.4 Bloom / halación (color) y Phosphor glow (mono)

Bloom (0..1, color): blur gaussiano de los píxeles más brillantes por encima de un umbral. Modela la dispersión del haz en el cañón cuando se sobrecarga. Realista en CRTs consumer: 0.2–0.5. En PVMs broadcast: 0.05–0.15 (cañón mejor controlado).

Halation (0..1, color): blur extra por canal — el rojo viaja más lejos que el verde/azul porque su fósforo decae más lento y porque la dispersión interna del cristal es asimétrica por cromaticidad. Crea ese halo rojizo alrededor de blancos puros. Realista: 0.1–0.4.

Phosphor glow (0..1, mono): equivalente unificado para monocromos. Un solo slider porque el canal es único.

8.5 Persistence per-channel (color) y single afterglow (mono)

Persistence R/G/B (0..5 ms, color): tiempo de decaimiento del fósforo por canal. R del P22 ≈ 1.0 ms; G ≈ 2.0 ms; B ≈ 1.0 ms (los datos exactos están en profiles/crts/<id>.json → phosphor.decay_ms). Realista: usar los valores del perfil cargado.

Por qué importa la asimetría: cuando algo blanco se mueve rápido (un coche en MotoGP, un bullet hell), la estela del frame anterior se compone con el nuevo. Como R decae más lento, la estela tiene tinte rojizo. Es el motion blur característico CRT — no se reproduce en paneles LCD/OLED si no es por shader explícito.

Afterglow (0..1, mono): equivalente single-channel. Slider único para terminales/osciloscopios. P1 (oscilo): 0.6–0.9 (estela larga, casi fosforescente). P31 terminal: 0.1–0.3.

8.6 Composition (barrel + vignette + gamma + aspect + bezel)

Barrel (0..0.3): distorsión radial de la imagen. 0 = plano (LCD). 0.05 = leve (PVM broadcast). 0.15 = TV consumer. 0.25 = arcade vertical. Realista en cada perfil: ya viene cableado al baseline.

Vignette (0..1): caída de brillo hacia esquinas. 0 = sin caída. 0.4 = típico PVM. 0.7 = arcade pronunciado.

Gamma display (1.8..2.6): corrección de salida (separada del gamma CRT). Modifica cómo se mapean tus colores de salida al panel donde Tubelight muestra. Si tu panel es sRGB calibrado, 2.2 está bien. Si tu panel sobre-satura, baja a 2.0.

Aspect ratio: Fill / 4:3 / 5:4 / 16:10 / 16:9 / 21:9. Snap-to-aspect en el menú: al elegir un ratio distinto de Fill, la ventana se redimensiona automáticamente para mantener píxeles cuadrados, sin letterbox.

Bezel (combo): off / soft / hard. Soft = sombra suave en bordes (sutil). Hard = recorte aristado. PNG photoreal queda diferido a v1.1.

8.7 Pass toggles — las 8 pasadas del pipeline

Cada pasada del pipeline OpenGL se puede desactivar individualmente con Ctrl+Alt+1..8 o desde el menú. Permite ver qué aporta cada una.

#NombreQué hace
−1signalYIQ-BW limit + dot crawl + ringing + ghosting + noise
0dither detectmáscara de patrones de dithering en alpha
1dither reconstructmezcla pares A/B para reconstruir gradiente
2beam + scanlineshaz horizontal + líneas de scan
3maskmáscara de fósforo (aperture/shadow/slot/etc.)
4bloom + halaciónblur umbral + per-channel halo

Ctrl+Alt+0 reactiva todas tras tener algunas off.

Pass 5 (temporal) está como identity en v0.1.0 — la persistencia se modela en pass 4 pero el ping-pong FBO frame-a-frame real está diferido. En v1.1 dejará estelas con decay correcto incluso entre frames.

8.8 Guardar como preset personalizado

El botón "Save preset" en el menú Image vuelca los valores actuales (perfil + signal + todos los sliders) a %APPDATA%\Tubelight\profiles\crts\<nombre>.json. Aparece en el dropdown CRT como [custom] <nombre> al lanzar Tubelight.

Para compartir: copiar ese JSON y mandarlo. El receptor lo coloca en su %APPDATA%\Tubelight\profiles\crts\ y le aparece como custom.

Formato: mismo schema que los bundle (schemas/crt_profile.schema.json). Puedes editarlos a mano con cualquier editor de texto.

9. Capturas y grabación

Tres mecanismos: screenshot PNG, vídeo MP4 con audio CRT opcional, modo recordable para que apps externas (Snipping Tool, Game Bar, OBS) puedan capturar el overlay.

9.1 Screenshot PNG (Ctrl+Alt+S)

Guarda el framebuffer post-pipeline en %USERPROFILE%\Pictures\Tubelight\tubelight-YYYYMMDD-HHMMSS.png (o en la carpeta que hayas configurado). El encode PNG corre en thread detached para no congelar el main loop 1-2 s a 1920×1200.

La imagen guardada es lo que ves — el contenido procesado, no la fuente cruda.

Toast en lower-left confirma con la ruta del PNG.

9.2 Vídeo MP4 (Ctrl+Alt+V)

Toggle. Primera pulsación inicia grabación, segunda la detiene. Requiere ffmpeg en PATH. Tubelight pipea raw RGB24 60 FPS al stdin de ffmpeg con args -vf vflip y output H.264 / yuv420p. Bitrate ~10 Mbps; ~30-60 MB por minuto.

Durante grabación, REC dot rojo parpadeando upper-left. Para parar, Ctrl+Alt+V otra vez. Toast con la ruta del MP4.

Si ffmpeg no está: el primer Ctrl+Alt+V muestra un toast "ffmpeg not found in PATH" y deshabilita futuras pulsaciones hasta que lo instales y reinicies.

9.3 Record source

Combo en tab Capture. Define qué pinta dentro del MP4: - overlay (default): el contenido procesado por Tubelight, mismo tamaño que la ventana overlay. - full monitor: la pantalla entera del monitor 0 incluyendo lo que rodea al overlay. - custom rect: un rectángulo arbitrario en coordenadas de monitor (campo de texto x,y,w,h en el menú).

El modo full monitor y custom rect usan BGRA del DXGI Duplication directamente (sin pasar por el pipeline) — útil para grabar áreas que el overlay no procesa.

9.4 Recordable (always-on desde v0.1.4)

Desde v0.1.4 (ADR-0001) Tubelight es siempre grabable por apps externas (Snipping Tool, Game Bar, OBS Display Capture) sin necesidad de pulsar ningún atajo. Antes (v0.1.3-) había un modo opt-in Ctrl+Alt+R que se eliminó porque añadía complejidad sin beneficio. La complejidad real está en por qué es necesario:

En Windows 10/11 con DWM, todas las APIs de captura desktop-level comparten un único bit binario de exclusión (WDA_EXCLUDEFROMCAPTURE). Por defecto Tubelight lo tiene ON para evitar feedback loop (si Tubelight no se excluye, DXGI Duplication captura su propio output → cadena CRT atenúa → colapso a negro tras pocos frames).

Ctrl+Alt+R hace dos cosas atómicas: 1. Quita WDA_EXCLUDEFROMCAPTURE → recorders externos ven el overlay. 2. Cambia la captura INTERNA de Tubelight de DXGI a Magnification API con filtro MagSetWindowFilterList(MW_FILTERMODE_EXCLUDE, [overlay_hwnd]). Mag es la única API Win32 que provee per-capturer exclusion (es pre-DWM, accesibilidad). Así Tubelight ignora su propio HWND y no hay feedback.

Límites: la API Mag está marcada "deprecated since Win10" en documentación de Microsoft (sigue funcional en Win11 26200; si MS la pula, migrar a la sucesora). El framerate efectivo de la Mag callback es 30-60 Hz dependiendo del driver. Recordable mode es per-session y no persiste (evita el problema de arrancar con overlay congelado por una mala combinación de settings).

Cómo grabar con Snipping Tool ahora: 1. Lanza Tubelight overlay encima de tu juego. 2. Win+Shift+S → selecciona la región del overlay → la captura sale procesada por CRT. Sin atajo previo.

9.5 Carpeta de capturas + Browse picker

Tab Capture → campo de texto con la ruta + botón Browse → file picker nativo Windows (IFileOpenDialog). Apply commits el cambio. Default vuelve a %USERPROFILE%\Pictures\Tubelight\.

La carpeta se crea si no existe. Persistencia en %APPDATA%\Tubelight\settings.json.

10. Audio CRT

Tubelight genera audio CRT realista vía XAudio2 (Win32 nativo). Por defecto OFF; activar en tab Audio.

Flyback whine: ~15.7 kHz (NTSC) o 15.6 kHz (PAL), modulado por luminancia media del frame. Es el silbido agudo característico de los CRTs en marcha. Imperceptible para muchos adultos (audición >40 años suele tener cutoff < 14 kHz); muy presente en grabaciones con micros de campo. Audibilidad alta en menores de 30 con audífonos sensibles.

Modulación por luminancia: cuando el frame medio es brillante, el cañón consume más → el flyback transformer trabaja más → whine más fuerte. Tubelight lee el luminancia media del último frame y modula amplitud ±20%.

Degauss thump: cuando cambias de perfil CRT, suena un thump grave + click de relay (~80 ms total). Modela el ciclo de desmagnetización que un CRT real ejecuta al encenderse. Toggle separado.

12. Casos de uso típicos

Recipe-book por tipo de contenido. Cada combinación incluye perfil + signal recomendados, aspect ratio sugerido, y notas opcionales.

12.1 Emuladores standalone

Mednafen (PCE / NES / SNES / SMS / GG / Saturn / PSX): perfil pvm-8220 + signal composite_ntsc para mercado US, commodore-1084s + composite_pal para PAL. Aspect: 4:3. Modo: --overlay-target Mednafen o --overlay-target-pid <PID>.

MAME (arcade vertical o horizontal): perfil wells-gardner-k7000 + signal composite_ntsc (US arcades) o rgb_vga (CRTs PC para Neo Geo en MVS). Aspect: depende del juego (3:4 vertical, 4:3 horizontal). Modo: target o region.

openMSX: perfil commodore-1084s (consumer monitors lo más parecido al setup MSX europeo) + signal composite_pal. Aspect: 4:3.

BlastEm (Mega Drive): perfil pvm-20m4 (broadcast 20") + signal composite_ntsc. Mega Drive sacaba composite por defecto; algunos sets podían sacar RGB SCART en PAL. Aspect: 4:3.

FCEUX / Nestopia (NES): perfil pvm-8220 + signal composite_ntsc. La NES sacaba composite muy ruidoso; si quieres autenticidad total, usa signal rf (lo que veían los niños 1985 con la TV conectada por antena).

Snes9x (SNES): perfil pvm-8220 + signal svideo. La SNES sacaba S-Video nativo si tenías el cable correcto. Aspect: 4:3 (algunos juegos PAL son 5:4).

12.2 RetroArch / libretro

Opción A — overlay encima de RetroArch: lanzar RetroArch sin shaders, lanzar Tubelight en target mode encima. Funciona pero pierdes integración con state save shaders de RetroArch.

Opción B — exportar preset .slangp desde Tubelight para cargarlo en RetroArch:

tubelight --export-slangp my-pvm.slangp --profile pvm-8220 --signal composite_ntsc

Copia el .slangp y los .slang shaders generados a la carpeta shaders/ de RetroArch y carga el preset desde Settings → Video → Shaders → Load Preset. Esta opción es menos fiel que el overlay (RetroArch ejecuta los shaders en su pipeline propio, no el de Tubelight) pero se integra con savestate, threaded video, etc.

13. Solución de problemas

Los problemas más comunes y cómo resolverlos. Si tu caso no está aquí, abre un issue en GitHub.

13.1 Overlay negro / no actualiza

Síntoma: la ventana de Tubelight se ve negra o congelada en el primer frame.

Causa típica 1: modo recordable activado previamente + WDA dropped + DXGI loop. v0.1.0 fuerza recordable OFF al arrancar para corregir esto. Si tienes un build viejo, edita %APPDATA%\Tubelight\settings.json y pon "recordable": false.

Causa 2: otro overlay encima (Discord overlay, Nvidia Shadowplay, Steam overlay). DWM compone primero y Tubelight ve el composite de todos. Desactiva esos overlays.

Causa 3: driver GL viejo. Tubelight necesita OpenGL 4.5 core. Actualiza el driver GPU.

13.2 Lag percibido en lo de debajo

Síntoma: el juego/terminal de debajo va con un frame de retraso.

Causa: Tubelight añade ~1 frame DXGI + ~1 frame compositing. A 60 Hz = ~33 ms. Es esperado.

Mitigación: activa low_latency en tab Image (vsync OFF + cap 240 FPS). Latencia baja a ~4-8 ms a costa de más uso de GPU. Cuidado: si pones low_latency=true y luego desactivas el cap manualmente, el main loop spineará a miles de FPS, saturando CPU+GPU. v0.1.0 fuerza cap 240 si low_latency está ON.

13.3 "ffmpeg not found" al grabar vídeo

Síntoma: Ctrl+Alt+V muestra toast ffmpeg not found in PATH.

Fix: instala ffmpeg via winget (winget install Gyan.FFmpeg), chocolatey o descarga manual desde gyan.dev. Reinicia Tubelight tras instalar (PATH se cachea por proceso).

PS> winget install Gyan.FFmpeg

13.4 Profile no carga

Síntoma: dropdown CRT cargas un perfil y nada cambia / toast con error de validación.

Fix: validar el JSON con tubelight --validate-profile profiles/crts/<id>.json. Te dice exactamente qué campo está mal o no cumple el schema. Comparar contra un bundle perfil para ver qué pinta tiene un JSON correcto.

> tubelight --validate-profile profiles/crts/pvm-8220.json

13.5 STATUS_STACK_BUFFER_OVERRUN al activar recordable (legacy)

Síntoma: en builds pre-v0.1.0 final, activar recordable mode crasheaba la app con /GS stack-canary fail (STATUS_STACK_BUFFER_OVERRUN).

Fix: v0.1.0 final lo arregló moviendo el MagCapture a heap-allocated unique_ptr (estaba stack-allocated junto al canary), llamando cpu_buffer_.resize() ANTES de MagSetImageScalingCallback, añadiendo destructor que llama shutdown() y haciendo filter_hwnds_[1] miembro persistente. Si estás en build viejo, actualiza.

13.6 Win+Shift+S no muestra overlay — checklist

  1. ¿Activaste Ctrl+Alt+R (recordable mode)? Sin él, Snipping Tool ve la app de debajo, no el overlay.
  2. ¿El toast confirmó "Recordable mode ON"?
  3. ¿Tubelight está realmente encima del área que estás capturando? Mueve la ventana o usa overlay-fullscreen.
  4. Si todo OK y aún sale el contenido sin procesar: cierra Snipping Tool, espera 2 s, vuelve a intentar (la app cachea estado).

13.7 Game Bar no graba overlay — checklist

Mismo problema y misma solución que §13.6 — Game Bar captura por DXGI Desktop Duplication o Windows.Graphics.Capture; ambos respetan WDA_EXCLUDEFROMCAPTURE.

Fix: activa Ctrl+Alt+R antes de empezar a grabar con Game Bar (Win+G → Capture → Record).

13.8 OBS sí graba (Window Capture WGC) — instrucciones

OBS Studio 28+ con la fuente "Window Capture" en modo WGC (Windows Graphics Capture) funciona con recordable mode. Pasos:

  1. Activa Ctrl+Alt+R en Tubelight.
  2. En OBS: Source → Add → Window Capture.
  3. Window: selecciona la ventana de Tubelight (Tubelight 0.1.0 — overlay).
  4. Capture Method: Windows 10 (1903 and up).
  5. Verifica que sale el contenido procesado en el preview.

Nota: "Display Capture" (la fuente que captura el escritorio entero) tampoco verá Tubelight a no ser que esté recordable mode ON.

14. Glosario

Aperture grille

Estructura de máscara CRT formada por hilos verticales finos (Trinitron). Da color brillante y verticales nítidas, a costa de horizontales más blandas y dos finos hilos de amortiguación visibles a contraluz.

Shadow mask

Lámina perforada situada detrás del fósforo en CRTs convencionales. Cada triada R-G-B comparte un agujero; el patrón se ve como un rejilla de puntos en zonas brillantes.

Slot mask

Variante a medio camino entre shadow mask y aperture grille: ranuras verticales en lugar de puntos o hilos. Típico de televisores y monitores de los 80–90 (Commodore 1084, muchos PVM consumer).

P22

Fósforo estándar de TV color en Europa y EE. UU. Tres componentes (R, G, B) con decaimientos relativos distintos: el rojo persiste más que verde/azul, lo que produce el característico arrastre rojizo en movimiento rápido.

P31

Fósforo verde estándar (Apple //e, VT100, osciloscopios). Persistencia media, pico en 525 nm. Cromaticidad muy saturada hacia el verde puro.

P3

Fósforo ámbar usado en terminales IBM 5151 y WYSE. Cromaticidad cálida (R alta, G media), reduce fatiga visual en sesiones largas según los criterios ergonómicos de los 80.

P4

Fósforo blanco de TV blanco y negro. Equilibrio cercano a blanco-azulado natural. Aparece en TVs B/N y Macintosh Classic.

P1

Fósforo verde de osciloscopio, persistencia larga. Hace que un trazo rápido deje estela visible muchos milisegundos.

Persistencia

Tiempo durante el cual el fósforo sigue emitiendo luz después de que el haz pase por encima. Determina el arrastre del movimiento (motion trail). Por canal: el R lento del P22 genera la estela rojiza típica.

Scanline

Cada una de las líneas horizontales pintadas por el haz CRT en cada frame. NTSC: 240 visibles, PAL: 288, VGA: 480.

Dot pitch

Distancia entre tríadas de fósforo de la máscara (en mm). Más fino = más resolución efectiva. PVM-8220: 0.30 mm; BVM-20F1U: 0.31 mm Super Fine Pitch; NEC 4FG: 0.28 mm.

Gamma CRT

Curva no lineal del cañón del CRT (≈2.2–2.5). Convierte el voltaje del haz en luminancia. Distinto del gamma display moderno (sRGB ≈ 2.2). Tubelight la modela por separado de la corrección de salida.

Trinitron

Tecnología Sony de aperture grille, dominante en monitores profesionales y de gaming desde los 80. Visible: imagen brillante, verticales nítidas, dos hilos de amortiguación cruzando horizontalmente.

Halation

Halo difuso alrededor de zonas brillantes causado por reflexión interna del vidrio y dispersión en el fósforo. Distinto al bloom: la halation es más roja porque el R viaja más lejos.

Bloom

Sangrado de luminancia en zonas saturadas: los píxeles brillantes parecen crecer en tamaño. En CRT real lo causa la dispersión del haz; en Tubelight es un blur gaussiano umbral-dependiente.

Barrel

Curvatura geométrica de la imagen, más pronunciada en TVs antiguos y arcade. Tubelight la aplica como distorsión radial sutil (configurable).

Viñeta

Caída de brillo hacia las esquinas. En CRT físico la causa la geometría del tubo y el bezel. Sutil en PVMs, muy marcada en arcades.

Bezel

Marco físico del CRT. Tubelight v0.1.0 lo omite por defecto (overlay quiere ser el cristal, no el mueble). Soporte de bezel PNG planeado para v1.1.

WDA_EXCLUDEFROMCAPTURE

Flag de DWM (Win10 build 2004+) que excluye una ventana de TODOS los capturadores desktop-level (Snipping Tool, Game Bar, OBS Display Capture). Es binario, no per-capturer. Tubelight la usa por defecto para evitar feedback loop al capturar lo que tiene debajo.

Magnification API

API Win32 de accesibilidad pre-DWM, única vía documentada para excluir UNA ventana concreta de SU PROPIO capturador (no del resto del sistema). El modo recordable de Tubelight la usa.

DXGI Desktop Duplication

API de DirectX para capturar el composite del escritorio DWM con latencia muy baja (~1 frame). Tubelight la usa para extraer el contenido que va a procesar el pipeline. Funciona con cualquier app (DX/GL/Vulkan) porque captura el composite final de DWM.

Posterize

Reducción del número de niveles tonales. Tubelight lo usa para emular pantallas 1-bit (Mac Classic = 2 niveles), terminales (6) y oscilo de fósforo P1. 0 = analógico (sin posterización).

Voltage bloom

Aumento dinámico del beam-width cuando el frame medio es muy brillante: el cañón se sobrecarga, el haz se ensancha, la imagen se ablanda. Real en CRTs físicos; pendiente como efecto opcional en v1.1.

Primarios SMPTE-C

Coordenadas xy de los fósforos R/G/B definidos por SMPTE para NTSC profesional 525-line. Distintos de Rec.601, EBU PAL y de los fósforos reales de cada CRT.

DWM

Desktop Window Manager: el compositor del escritorio de Windows desde Vista. Renderiza cada ventana a su propio buffer y compone el resultado con efectos. Toda captura desktop-level (excepto Magnification API) pasa por DWM, lo que tiene implicaciones para el modo recordable.

1. Welcome

Tubelight is a CRT overlay that sits on top of any Windows application and processes whatever it shows — a retro game, an emulator, a terminal, a video player, anything — so it looks like a real tube monitor: scanlines, phosphor mask, glass halation, red persistence, subtle beam blooming.

It is not a capture filter or a shader bolted onto an emulator. It is a real Windows window that captures the desktop underneath via DXGI Desktop Duplication, applies an 8-pass OpenGL pipeline in real time, and displays the result at 60 FPS on top. Any app running below — DirectX, OpenGL, Vulkan, pure software — comes out rendered through the tube.

The design word is fidelity: each of the 16 CRT profiles is calibrated against an actual factory service manual, with phosphor chromaticity cited, verified dot pitch, and per-channel persistence. It is not "a generic CRT with sliders": it is Sony PVM-8220 vs. Sony BVM-20F1U vs. Sony FW900 vs. Sharp X68000 CZ-602D, each with its specific look.

What Tubelight is, in one page

What Tubelight does: paints a realistic CRT glass on top of your screen, calibrated against historical hardware. Four use modes, 16 bundled tube monitors, 7 signal profiles (RF, composite NTSC/PAL, S-Video, SCART RGB, Component, RGB VGA), 8-pass OpenGL pipeline, PNG screenshots and MP4 video via ffmpeg, CRT audio via XAudio2.

What it does NOT do: emulate hardware. Tubelight knows nothing about games, ROMs, emulators or actual video signals — it just grabs the already-rendered image from another app and transforms it. It is not a replacement for RetroArch / Mednafen / openMSX, but a companion that adds the monitor those apps lack.

Why the distinction matters: many "CRT filters" add scanlines as a multiply layer. Tubelight models phosphor chromaticity, mask, per-channel persistence, geometric distortion and halation separately, and each profile carries the real values of the monitor it imitates. The difference between a PVM-8220 (aperture grille, standard P22) and a BVM-20F1U (Super Fine Pitch aperture grille, broadcast-calibrated P22) is visible — because it comes from the service manuals.

Use cases

Standalone emulators: Mednafen, MAME, BlastEm, openMSX, FCEUX. Launch them, then launch Tubelight on top in windowed or target mode, pick a profile → the emulator output goes through the tube even though the emulator has no shader at all.

libretro / RetroArch emulators: for in-runner integration, export the current profile as a .slangp preset with tubelight --export-slangp out.slangp --profile X --signal Y and load it in RetroArch. Or just run the overlay on top and skip libretro entirely.

Modern retro games: UFO 50, Sea of Stars, Shovel Knight, Children of Morta, Crypt of the NecroDancer. Pixel art on a flat panel never gives the original feel; route it through a pvm-8220 or commodore-1084s and the pixels "breathe" the way they should.

Video players: mpv, VLC, browser, YouTube. Useful for archival material (VHS rips, demos, 80s music videos). The tv-bw-p4 profile with rf signal turns any modern clip into a 1975 tube TV.

Terminals and code: cmd, PowerShell, WSL, VS Code. terminal-p31 (Apple //e green) or terminal-p3-amber (IBM 5151 amber) over your editor gives real vintage ergonomics without giving up modern tooling.

Demo / streaming video: with recordable mode (Ctrl+Alt+R), apps like Snipping Tool, Game Bar and OBS can record the processed overlay (not the raw source). Useful for showing your setup as it appears on screen.

Non-goals (what Tubelight does NOT do, on purpose)

  • It is not an emulator. If you want to play SMW, you need Snes9x; Tubelight paints on top.
  • It does not generate signal. No internal pattern generator, no 31 kHz / 15 kHz sync. If the app underneath runs at 144 Hz, Tubelight processes at 144 Hz (does not downsample to 60).
  • It does not remove lag. The pipeline adds ~1 frame for DXGI + ~1 frame for composition. If competitive-gaming latency matters, low_latency mode minimizes it (vsync off + 240 FPS cap), but it is still an overlay, not a wrapper.
  • It does not record the underlying app with synced audio. Ctrl+Alt+V records the processed overlay to MP4 H.264 (video only). For game audio use OBS with Window Capture WGC (compatible with recordable mode).
  • It does not emulate consoles, mappers, chip audio, or anything else your emulator does. Just the monitor.
  • It does not persist config across user accounts. Each user has their own %APPDATA%\Tubelight\settings.json.
  • It is not portable. Requires Windows 10 build 2004+ / 11 for modern DXGI Desktop Duplication + DWM composite. Linux planned for v1.1 (no recordable mode; use OBS-Wayland or gpu-screen-recorder).

2. Installation

Tubelight v0.1.3 (Windows; Linux coming soon in v1.1) ships as a single tubelight.exe (~1.2 MB) plus runtime DLLs (epoxy-0.dll, glfw3.dll) and the profiles/ folder with the 16 CRT + 7 signal profiles. No installer needed; copy the folder anywhere and run the executable.

Double-clicking tubelight.exe launches the overlay in windowed mode with the last used profile (or pvm-8220 / composite_ntsc on first launch). The binary opens a resizable Win32 window that displays the desktop area underneath, processed by the pipeline.

Requirements

  • OS: Windows 10 build 2004 (May 2020) or later; Windows 11 recommended.
  • GPU: any with OpenGL 4.5 core profile support (Intel HD 4400 / 2014, NVIDIA Kepler / 2012, AMD GCN 1.0 / 2012 onwards). Any modern iGPU is enough.
  • CPU: one free core. The pipeline runs on the GPU; the CPU only presents frames and moves data through DXGI.
  • RAM: 200 MB typical.
  • Disk: 5 MB for the executable + DLLs + profiles. PNG captures ~2 MB each at 1920×1200, MP4 video ~30-60 MB/min at 60 FPS.
  • ffmpeg (optional): for Ctrl+Alt+V MP4 recording. Must be on PATH. If absent, the rest works — recording is disabled with a clear toast.
  • Magnification API: bundled with Windows since XP. Only used in recordable mode; no install needed.

Download and install

Option A — release zip (recommended): download the latest tubelight-<version>-win64.zip from the Releases page, unzip into C:\Tools\Tubelight\ (or wherever you like). Double-click tubelight.exe.

Option B — local build from source: requires Visual Studio 2022 BuildTools (with C++), vcpkg, CMake 3.26+. Run scripts\build_windows.bat from the repo root. The build produces build\windows-vcpkg\Release\tubelight.exe and copies DLLs + profiles next to it. ~3-5 min cold, ~1 min incremental.

First run creates %APPDATA%\Tubelight\settings.json with defaults. Screenshots go to %USERPROFILE%\Pictures\Tubelight\ by default; change it in the Capture tab → Browse.

Build local Windows
> scripts\build_windows.bat
Verificar versión
> tubelight --version

First run

Double-click tubelight.exe. A movable 1280×960 window appears. What you see inside is the desktop area underneath the window, processed by the pipeline (default profile pvm-8220 + signal composite_ntsc).

Drag the window over whatever you want to process — an emulator, a video, a terminal. Drag the edges to resize; the pipeline reconfigures FBOs automatically.

Ctrl+Alt+M opens the in-app menu (five tabs: Profile / Image / Capture / Audio / Help). Profile picks the CRT monitor and signal. Image tunes scanlines, mask, bloom, persistence. Capture sets the screenshot folder. Audio handles the flyback whine. Help lists shortcuts and credits.

Ctrl+Alt+Q quits Tubelight.

Uninstall / portability

Tubelight does not touch the Windows registry. To uninstall:

  1. Delete the folder containing tubelight.exe.
  2. Delete %APPDATA%\Tubelight\ (settings + user presets).
  3. Optional: delete %USERPROFILE%\Pictures\Tubelight\ (captures and videos).

For backup: zip the binary folder + %APPDATA%\Tubelight\. Restore by unzipping where they were.

3. Interface tour

Tubelight has four visual elements the user interacts with: the overlay window, the in-app menu (five tabs), the status HUD (top-right corner, optional) and the toast notifications (bottom-left, ephemeral).

INFO: The menu is built with Dear ImGui. If it grows too large for your screen, drag its title bar to move it. Its positions don't persist yet (planned for v1.1).

Overlay window

Default: resizable framed Win32 window, 1280×960. Shows the desktop content underneath, already processed.

  • Drag the title bar to move.
  • Drag any edge to resize. During the drag, the pipeline keeps processing (no frozen final-frame as in many GLFW apps).
  • Double-click the title bar: maximize / restore.
  • Ctrl+Alt+Enter: toggle runtime fullscreen preserving the chosen aspect ratio (no GWL_STYLE changes, see §13 on the avoided bug).

In-app menu (5 tabs)

Ctrl+Alt+M opens or closes it. Dark UI with a 6-accent palette per category (sky / teal / amber / coral / lavender / mint) and parchment cream+gold tooltips explaining each widget in plain Spanish.

Profile tab (sky): CRT dropdown (16) + Signal dropdown (7) + Global intensity slider 0..2.0× over the profile baseline. When changing profile, the Image-tab sliders jump to the new profile baseline values.

Image tab (teal): groups for Scanlines / Beam, Phosphor mask, Phosphor colour (mono), Bloom / Halation, Persistence, Composition (barrel, vignette, display gamma, aspect ratio, bezel), Pass toggles (8 checkboxes). Distinct layout for color CRTs vs. monochromes (see §8).

Audio tab (coral): master volume + flyback whine toggle + degauss thump toggle on profile change.

Help tab (lavender + mint): shortcuts table, credits, link to this manual, version + git SHA.

Status HUD and toasts

HUD (Ctrl+Alt+H): top-right corner. Shows current profile / signal / mode (windowed / fullscreen / target / region) in small mono on a translucent background. Persists until toggled off.

Toasts: 3 s ephemeral notifications in the bottom-left corner. Appear on screenshot save, recording start/stop, profile change, ffmpeg error, recordable mode activation. Do not block input.

REC dot: blinking red dot in the top-left while video is recording. Independent of toasts.

4. Your first CRT overlay

A 60-second walkthrough to see the effect on your own desktop.

  1. Open anything pixel-art or text-heavy: an emulator with a game loaded, a terminal with green text, a screenshot in Windows Photos, whatever you have handy. It needs to take up enough space for a 1280×960 overlay to cover it.
  1. Double-click tubelight.exe. The overlay window appears, probably centered on your primary monitor; drag it on top of your content.
  1. Press Ctrl+Alt+M to open the menu. In the Profile tab (sky blue icon), open the CRT dropdown. Start with something familiar:
  2. - pvm-8220 for a common 80s–90s Sony pro monitor.
  3. - commodore-1084s for an 80s consumer slot-mask TV/monitor with that Amiga and home-computer feel.
  4. - terminal-p31 if what you have underneath is text and you want 70s green.
  5. - mac-classic-white for pure bluish-white 1-bit (yes, posterize to 2 levels, yes, it looks just like a Mac SE in 1989).
  1. In the same tab, open the Signal dropdown. For retro console games → composite_ntsc (what your Mega Drive emitted over the yellow RCA). For terminal → rgb_vga (clean, pixel-perfect). To feel old-TV noise → rf.
  1. Look at what's underneath. Scanlines are visible, pixels are no longer flat bricks, white has a halo, reds have a minimal trail when something moves.

5. Overlay modes

Four modes, all available from the CLI or from the menu once launched. Pick based on the case.

Windowed (default)

Resizable framed Win32 window. Default mode on double-click. Useful to keep Tubelight visible while you work on something else next to it. Supports drag, resize, click-through, snap-to-aspect, and all shortcuts.

CLI: tubelight --overlay [--size W H] [--profile X --signal Y]

> tubelight --overlay --profile pvm-8220 --signal composite_ntsc --size 1280 960

Runtime fullscreen (Ctrl+Alt+Enter, IF-safe)

Borderless, full monitor, on top of everything. Preserves the aspect ratio picked in the Image tab (4:3, 5:4, 16:10, 16:9, 21:9, Fill).

Implementation: WM_NCCALCSIZE override that collapses title and borders to 0 px without touching GWL_STYLE, plus SetWindowPos with monitor.height − 1 to keep the 1-px transparent bottom line that blocks Win11 Independent Flip and keeps the window on the DWM composited path (required for WDA_EXCLUDEFROMCAPTURE to apply). See §13 Troubleshooting and the lessons learned in docs/manual/ for why this isn't done the "obvious" way.

From CLI: tubelight --overlay-fullscreen [--profile X --signal Y] starts directly without windowed.

From overlay: Ctrl+Alt+Enter toggles at runtime preserving aspect.

> tubelight --overlay-fullscreen --profile pvm-8220 --signal composite_ntsc

Target window (Ctrl+Alt+T)

Pins Tubelight to a specific window. The window moves, Tubelight follows every frame. The window resizes, Tubelight resizes. The window minimizes, Tubelight hides.

Designed for standalone emulators (Mednafen, openMSX, MAME, FCEUX). Launch them without shaders, then launch Tubelight on top in target mode → they become PVMs or whatever you pick.

Automatic click-through: the target keeps receiving input. Does not interfere with the game's keyboard or mouse.

Two ways to target: title substring or PID. PID is more robust (no confusion with other windows).

> tubelight --overlay-target "Mednafen" --profile pvm-8220 --signal composite_ntsc

> tubelight --overlay-target-pid 12345 --profile sharp-x68k-cz602d --signal rgb_vga

Fixed region (--overlay-region)

Paints on a fixed monitor rectangle without following any window. Useful for keeping a static CRT area (e.g. "this slice of my setup is always Sega Mega Drive on composite NTSC") even when the apps underneath change.

Click-through always on. Takes no input.

Coordinates: x,y = top-left corner; w,h = width and height. In pixels of monitor 0 by default.

> tubelight --overlay-region 200,150,1280,960 --profile commodore-1084s --signal composite_pal

6. CRT profiles — catalog of the 16 monitors

Tubelight ships 16 profiles. Each one carries mask type, dot pitch, diagonal, aspect, sweep frequencies, phosphor chromaticity and per-channel persistence — all verified against a service manual or equivalent source (crtdatabase, ManualsLib, archive.org, gamesx.com, Wikipedia EIA phosphor table). The concrete URLs are in each profile's JSON (profiles/crts/<id>.json → source.url field).

The gallery below is the same reference testcard processed by each profile with composite_ntsc signal. Switch the signal in §7 if you want to see how the chain varies inside the same monitor.

6.1 Sony PVM-8220 (Trinitron 8", 1985)

Small pro broadcast monitor, aperture grille (Trinitron), 0.30 mm dot pitch, 8" diagonal, native 4:3. Standard P22 phosphor, SMPTE-C primaries. Asymmetric per-channel persistence (slow R, fast GB → red trail).

Look: bright, crisp verticals (Trinitron), pronounced scanlines (240 visible on NTSC), two damping wires visible against bright fields crossing the frame horizontally. Most-used monitor in NTSC retro game testing for its color accuracy and fine dot pitch.

Source: crtdatabase.com/crts/sony/sony-pvm-8220 (specs), epanorama.net/documents/video/phosphor_decay.html (decay), crtdatabase.com/faq/phosphor-designations (SMPTE-C primaries).

6.2 Sony PVM-20M4 (Trinitron 20", 1996)

Mid-90s 20" Trinitron, 4:3, 0.31 mm dot pitch. Inherits P22 phosphor and SMPTE-C primaries from the PVM-8220 but with a larger tube and slightly more curved geometry. Scanlines at 240 NTSC are still visible but more subtle than in the 8" because each painted line is physically taller. Used in broadcast well into the 2000s.

6.3 Sony BVM-20F1U (Broadcast 20", Super Fine Pitch)

Broadcast Video Monitor: the pro line above the PVM. Super Fine Pitch (~0.31 mm but with more controlled grille geometry), factory-calibrated, SMPTE-C primaries verified by instrument. The monitor that defined how SMPTE C would look in a 1990s control room.

In Tubelight the difference vs. PVM-8220 is subtle: chromaticities nearly identical (same standard), but beam focus is higher and bloom barely exists, giving a more "clinical" image. Source: ManualsLib BVM Operation Manual + crtdatabase BVM specs.

6.4 Sony GDM-FW900 (Trinitron 24" 16:10, 1998)

The widescreen CRT holy grail. 24" Trinitron 16:10 (2304×1440 @ 80 Hz official; many units reach 2560×1600), aperture grille 0.23 mm at center (0.27 mm at edges, noted in JSON), PC-monitor primaries close to sRGB. Demoscene star 2000–2010.

Look: clean chromaticity, ultra-fine dot pitch (scanlines at 1080p barely visible because each line has many triads), very high beam focus. Tubelight assumes PC mode (no chroma subsampling), hence "clean" compared to a consumer PVM. Source: ManualsLib FW900 service manual.

6.5 Commodore 1084S (Slot mask 14", PAL/NTSC)

Commodore consumer-pro monitor, sold with Amiga 500/2000. Slot mask (not aperture grille), 0.45 mm dot pitch originally — Tubelight v0.1.0 corrects to 0.25 mm because the most reliable source (Service Manual 1084S-D1 on archive.org) gives 0.42 mm horizontal triad / 0.25 mm vertical slot. 14" diagonal, native PAL in EU market, NTSC in US.

Look: slot mask visible as a faint vertical grid, consumer P22 phosphor (slightly warmer than pro SMPTE-C), pronounced scanlines, generous bloom and halation from the bezel and thick glass. The Amiga 1985–1992 look when connected to a 1084. Source: Commodore 1084S-D1 Service Manual (archive.org).

6.6 Sharp X68000 CZ-602D (15" tube, 0.39 mm)

Original monitor Sharp shipped with the X68000 Compact/Super (1989–1991). 15" diagonal (corrected vs. earlier docs that said 14"), 0.39 mm dot pitch, conventional shadow mask, 24/31 kHz multisync, stereo audio. Warmer look than a PVM due to the Japanese consumer phosphor and the front-glass anti-glare filter.

Source: gamesx.com X68000 wiki + Sharp CZ-602D service manual (Japanese repro, archive.org).

6.7 Sharp X68000 CZ-603D (14", drops 24 kHz)

Smaller successor to the CZ-602D, 14", same phosphor and mask but without native 24 kHz mode (only 31 kHz). In Tubelight the difference shows as slightly finer scanlines and less curved geometry.

6.8 Sharp CZ-614D (15", multisync con audio estéreo)

Wider multisync version (15–38 kHz) aimed at the later X68000 line. More stable geometry than the previous two, same Japanese consumer phosphor. Stereo audio integrated into the monitor (no external RCA inputs).

6.9 NEC MultiSync (1986, JC-1401P3A)

The first multisync monitor in history (NEC, 1986). 14", conventional shadow mask, 0.31 mm dot pitch. Initially CGA/EGA/VGA compatible. P22 phosphor with chromaticity slightly shifted towards greens (NEC habit).

Source: archive.org/details/JC-1401P3A-MultiSync (original user manual).

6.10 NEC MultiSync 4FG (1990, 0.28 mm fine pitch)

1990 generation, 0.28 mm dot pitch — one of the finest of its time for a consumer monitor. SVGA compatible. Much cleaner image than the original MultiSync; less pronounced scanlines, more uniform phosphor.

Source: NEC MultiSync 4FG Service Manual (archive.org).

6.11 Wells-Gardner K7000 (arcade 13", 0.65 mm)

The arcade monitor par excellence, mid-80s to 2000s. 13" diagonal, 0.65 mm dot pitch (huge by modern standards — pixels are bricks, scanlines are walls). P22 phosphor with brutal saturation, medium beam focus, marked barrel geometry.

Look: the essence of vertical arcade. If you have MAME running Defender, Robotron, Pac-Man, etc., this is the profile.

6.12 Generic PVM (baseline 14")

Baseline profile. Doesn't imitate any specific monitor, serves as a neutral starting point to begin tweaking sliders without bias. Aperture grille 0.30 mm, P22 SMPTE-C, symmetric RGB persistence. If you want to dial in YOUR ideal values and save a preset, start here.

6.13 Terminal P31 green (Apple //e / VT100)

Green monochrome. P31 phosphor with chromaticity (0.10, 1.30, 0.20) — barely any red tint, brutal green dominance towards pure 525 nm green, minimal blue tint. Default posterize: 6 levels (characteristic of pre-greyscale terminals). Medium persistence: text scrolls with a barely perceptible trail.

Use with rgb_vga for a modern terminal, composite_ntsc if you emulate an Apple //e or Commodore PET on composite.

Source: Wikipedia EIA Phosphor Designations + Apple //e Owner's Manual.

6.14 Terminal P3 amber (IBM 5151 / WYSE)

Amber monochrome. P3 phosphor with chromaticity (1.40, 0.65, 0.05) — high R + medium G, almost no B. Warm tone designed to reduce eye fatigue over long sessions, per early-80s ergonomic guidance (German medical studies; some terminals shipped "ergonomically optimized" and sold ~30% more expensive).

Used in IBM 5151 (original PC mono), WYSE-50, various banking terminals. Default posterize: 6.

6.15 B&W TV P4 (vintage 17" TV)

1960s–70s B&W television. P4 bluish-white phosphor (chromaticity 0.95, 1.00, 1.10). Posterize 0 (pure analog, no posterization). More curved geometry (high barrel) than a pro monitor; generous vignette. Paired with rf signal gives the full 70s TV look.

6.16 Mac Classic (9", 1989, 1-bit P4)

The Macintosh Classic / SE / Plus monitor. 9", P4 bluish-white phosphor, posterize 2 levels (pure 1-bit: white or black, no greys). To see what Mac users saw in 1989–1993, almost like a pre-applied Floyd-Steinberg dither.

Only profile with such aggressive posterize. Used on color images, the effect is brutal: everything collapses to a black/white pattern. For B&W photos or 1-bit pixel art it works impeccably.

7. Signal profiles — the 7 chain paths

How video reached the monitor in its day. Each signal models a different set of physical artifacts: additive noise, chroma smear, dot crawl, ringing, ghosting, rainbow banding. Same monitor + different signal = radically different image.

The gallery uses a fixed PVM-8220 so the only variable is the signal.

7.1 RF (broadcast / antenna)

Worst-of-the-worst in quality — and the most nostalgic for many. Analog modulation to broadcast TV band (channel 3/4 or coax). In Tubelight: high additive noise, very visible dot crawl, pronounced horizontal ringing, mild ghosting. What you saw plugging the NES through an RF adapter into an old TV.

7.2 Composite NTSC (consumer RCA, NTSC 525-line)

Yellow RCA. A single line for multiplexed luma + chroma. Key models in Tubelight: chroma smear (color edges blur horizontally because chroma is sub-sampled), dot crawl (streaked pattern on saturated edges — the beat between carrier frequency and luma), moderate ringing.

What Mega Drive, SNES, NES, PlayStation, N64 emitted by default. For NTSC console games this is the signal your hardware actually saw.

7.3 Composite PAL (consumer, 625-line, EBU)

European equivalent. Chroma delay line (Phase Alternating Line) cancels phase inversions between adjacent lines → less rainbow banding than NTSC, more color stability. 625-line, 50 Hz. Tubelight models less dot crawl, similar chroma smear, scanlines at 288 visible.

7.4 S-Video (Y/C, separate luma + chroma)

Two pins: one for luma (Y), one for chroma (C). No multiplexing = no dot crawl, no horizontal chroma smear (still slight vertical subsampling). Same baseline noise as composite. Best quality-to-cost upgrade over composite — many hardcore retro folks ran S-Video cables for SNES (s-video by default) and Saturn.

7.5 SCART RGB (RGBS, almost clean)

Analog RGB + separate sync, traveling on SCART cable (European standard). No multiplexing, no chroma subsampling. Only the analog cable noise + the monitor's shadow/aperture-grille mask remain. The "clean" output of European retrogaming: BVM/PVM + SCART RGB cable was the definitive 90s setup.

7.6 Component (YPbPr, differential separation)

Y + Pb + Pr on three RCA cables. Differential luma/chroma separation → quality equivalent to SCART RGB with more practical cables. Used in US high-end consumer (PlayStation 2 component, Xbox component, certain DVDs).

7.7 RGB VGA (clean passthrough, pixel-perfect)

No significant analog artifacts. What you saw on a 90s+ PC monitor with a standard VGA cable. Tubelight applies this as a digital passthrough: zero noise, zero smear, zero ringing. The only distortions you see come from the CRT profile (mask, scanlines, phosphor, bloom) — useful to isolate the monitor contribution without signal coloration.

8. Fine tuning — anatomy of the Image menu

The Image tab is where you spend 80% of your time after picking a profile. Each slider models a physical CRT phenomenon. What follows goes section by section with the physics behind, practical ranges, and when each is useful.

8.1 Scanlines / Beam (4 sliders)

Scanline strength (0..1): how much the inter-line space darkens. 0 = no scanlines (flat panel). 1 = maximum. Realistic on consumer CRTs: 0.6–0.8. On fine broadcast monitors (BVM): 0.3–0.5 (because the line is better focused and almost covers the gap).

Beam width (0.5..2.0): beam thickness. <1 = thinner beam → scanlines more visible, image more "crisp". >1 = thicker beam → lines overlap, image softer.

Gamma (1.8..2.6): CRT gun gamma (not display gamma). Physically: the tube's own voltage→luminance curve, distinct from sRGB output correction. Standard EBU/SMPTE value: 2.2. Broadcast PVMs usually calibrate to 2.5 (the original NTSC curve). High values = deeper shadows, blacker black.

Scanline count: number of visible lines. 240 (NTSC), 288 (PAL), 480 (VGA). Determines how many scanlines the shader paints at your window resolution — independent of aspect ratio.

Common trap: scanline count != source resolution. If your image is 320×240 and you pick 480 scanlines, you see 480 lines but each pair shares the same source pixel. Apparent fix: use the real NTSC/PAL count (240/288) for retro console games.

8.2 Phosphor mask (color CRTs)

Color CRTs only (greyed-out on mono).

Type combo: 6 modeled mask types. - shadow_mask: R-G-B dot grid. NEC MultiSync, most consumer TVs. - aperture_grille: vertical grille (Trinitron). Crisp verticals. - slot_mask: vertical slots (Commodore 1084S). - diamond_mix: slot variant with diamond triads (some Hitachi). - none: no mask (debug). - auto: uses the loaded profile's.

Strength (0..1): how visible the mask is. 0 = invisible (interpolated). 1 = hard mask. Realistic values: 0.4–0.7. On fine-pitch monitors (0.28 mm): 0.2–0.4 (mask is physically smaller and less visible).

Pitch (0.20..0.65 mm): mask size. Changes pattern spacing.

8.3 Phosphor colour (monochromes)

Mono CRTs only (greyed-out on color).

Tint R/G/B (0..2.0): phosphor chromaticity. Defines the dominant color. For P31 green: (0.10, 1.30, 0.20). For P3 amber: (1.40, 0.65, 0.05). For P4 white: (0.95, 1.00, 1.10).

Presets: P31 green, P3 amber, P4 white, P1 oscilloscope, custom. Custom saves the values into the preset on export.

Posterize levels (0..16): tonal level count after phosphor application. 0 = analog (no posterization, continuous gradient). 2 = 1-bit (only two levels, Mac Classic). 6 = typical pre-greyscale terminal. 16 = quasi-continuous 4-bit. Drastically changes the feel.

8.4 Bloom / halation (color) and Phosphor glow (mono)

Bloom (0..1, color): Gaussian blur of pixels above a brightness threshold. Models beam dispersion when the gun overloads. Realistic on consumer CRTs: 0.2–0.5. On broadcast PVMs: 0.05–0.15 (better-controlled gun).

Halation (0..1, color): extra per-channel blur — red travels farther than green/blue because its phosphor decays slower and because internal glass dispersion is asymmetric by chromaticity. Creates that reddish halo around pure whites. Realistic: 0.1–0.4.

Phosphor glow (0..1, mono): unified equivalent for monochromes. One slider because the channel is single.

8.5 Per-channel persistence (color) and single afterglow (mono)

Persistence R/G/B (0..5 ms, color): per-channel phosphor decay time. P22 R ≈ 1.0 ms; G ≈ 2.0 ms; B ≈ 1.0 ms (exact data in profiles/crts/<id>.json → phosphor.decay_ms). Realistic: use the loaded profile's values.

Why asymmetry matters: when something white moves fast (a MotoGP bike, a bullet hell pattern), the previous frame's trail composes with the new one. As R decays slower, the trail has a red tint. The characteristic CRT motion blur — not reproduced on LCD/OLED panels without an explicit shader.

Afterglow (0..1, mono): single-channel equivalent. One slider for terminals/oscilloscopes. P1 (scope): 0.6–0.9 (long, almost phosphorescent trail). P31 terminal: 0.1–0.3.

8.6 Composition (barrel + vignette + gamma + aspect + bezel)

Barrel (0..0.3): radial image distortion. 0 = flat (LCD). 0.05 = mild (broadcast PVM). 0.15 = consumer TV. 0.25 = vertical arcade. Realistic value for each profile: already wired to baseline.

Vignette (0..1): brightness falloff toward corners. 0 = no falloff. 0.4 = typical PVM. 0.7 = pronounced arcade.

Display gamma (1.8..2.6): output correction (separate from CRT gamma). Affects how output colors map to the panel Tubelight displays on. If your panel is sRGB-calibrated, 2.2 is fine. If your panel over-saturates, drop to 2.0.

Aspect ratio: Fill / 4:3 / 5:4 / 16:10 / 16:9 / 21:9. Snap-to-aspect in the menu: picking a ratio other than Fill auto-resizes the window to keep square pixels, no letterbox.

Bezel (combo): off / soft / hard. Soft = subtle edge shadow. Hard = aristate clip. Photoreal PNG deferred to v1.1.

8.7 Pass toggles — the 8 pipeline passes

Each OpenGL pipeline pass can be toggled individually with Ctrl+Alt+1..8 or from the menu. Lets you see what each one contributes.

8.8 Save as custom preset

The "Save preset" button in the Image tab dumps current values (profile + signal + all sliders) to %APPDATA%\Tubelight\profiles\crts\<name>.json. Shows up in the CRT dropdown as [custom] <name> on next launch.

To share: copy that JSON and send it. Receiver drops it into their %APPDATA%\Tubelight\profiles\crts\ and it appears as custom.

Format: same schema as bundled (schemas/crt_profile.schema.json). Editable by hand with any text editor.

9. Captures and recording

Three mechanisms: PNG screenshot, MP4 video with optional CRT audio, recordable mode so external apps (Snipping Tool, Game Bar, OBS) can capture the overlay.

9.1 PNG screenshot (Ctrl+Alt+S)

Saves the post-pipeline framebuffer to %USERPROFILE%\Pictures\Tubelight\tubelight-YYYYMMDD-HHMMSS.png (or to the configured folder). PNG encoding runs in a detached thread to avoid freezing the main loop 1-2 s at 1920×1200.

The saved image is what you see — processed content, not the raw source.

Toast in lower-left confirms with the PNG path.

9.2 MP4 video (Ctrl+Alt+V)

Toggle. First press starts recording, second stops it. Requires ffmpeg on PATH. Tubelight pipes raw RGB24 60 FPS into ffmpeg stdin with args -vf vflip and H.264 / yuv420p output. Bitrate ~10 Mbps; ~30-60 MB per minute.

During recording, blinking red REC dot upper-left. To stop, Ctrl+Alt+V again. Toast with the MP4 path.

If ffmpeg is missing: the first Ctrl+Alt+V shows a toast "ffmpeg not found in PATH" and disables further presses until you install and restart.

9.3 Record source

Combo in the Capture tab. Defines what is painted into the MP4: - overlay (default): the Tubelight-processed content, same size as the overlay window. - full monitor: the entire monitor 0 screen including what surrounds the overlay. - custom rect: an arbitrary rectangle in monitor coordinates (x,y,w,h text field in the menu).

full monitor and custom rect use BGRA directly from DXGI Duplication (bypassing the pipeline) — useful for recording areas the overlay doesn't process.

9.4 Recordable (always-on since v0.1.4)

Opt-in mode (default OFF) that lets external apps (Snipping Tool, Game Bar, OBS Display Capture) see the processed overlay. The real complexity lies in why it's needed:

On Windows 10/11 with DWM, all desktop-level capture APIs share a single binary exclusion bit (WDA_EXCLUDEFROMCAPTURE). By default Tubelight has it ON to prevent a feedback loop (without exclusion, DXGI Duplication would capture Tubelight's own output → the CRT chain attenuates → collapses to black after a few frames).

Ctrl+Alt+R does two atomic things: 1. Removes WDA_EXCLUDEFROMCAPTURE → external recorders see the overlay. 2. Switches Tubelight's INTERNAL capture from DXGI to Magnification API with MagSetWindowFilterList(MW_FILTERMODE_EXCLUDE, [overlay_hwnd]). Mag is the only Win32 API that provides per-capturer exclusion (pre-DWM, accessibility). Tubelight ignores its own HWND, so no feedback.

Limits: the Mag API is marked "deprecated since Win10" in Microsoft docs (still functional in Win11 26200; if MS pulls it, migrate to the successor). Mag callback effective framerate is 30-60 Hz depending on the driver. Recordable mode is per-session and doesn't persist (avoids the problem of starting with a frozen overlay from a bad settings combo).

Recording with Snipping Tool now: 1. Launch Tubelight overlay on top of your game. 2. Win+Shift+S → select the overlay region → the capture comes out CRT-processed. No shortcut needed.

9.5 Captures folder + Browse picker

Capture tab → text field with path + Browse button → native Windows file picker (IFileOpenDialog). Apply commits the change. Default returns to %USERPROFILE%\Pictures\Tubelight\.

Folder is created if absent. Persisted in %APPDATA%\Tubelight\settings.json.

10. CRT audio

Tubelight generates realistic CRT audio via XAudio2 (native Win32). Default OFF; enable in the Audio tab.

Flyback whine: ~15.7 kHz (NTSC) or 15.6 kHz (PAL), modulated by the frame's average luminance. The characteristic CRT high-pitched whine. Inaudible to many adults (hearing >40 usually cuts off below 14 kHz); very present in field-mic recordings. High audibility for under-30s with sensitive headphones.

Luminance modulation: when the average frame is bright, the gun draws more current → the flyback transformer works harder → louder whine. Tubelight reads the average luminance of the last frame and modulates amplitude ±20%.

Degauss thump: when changing CRT profile, a low thump + relay click sounds (~80 ms total). Models the demagnetization cycle a real CRT runs on power-up. Independent toggle.

11. Keyboard shortcuts — full table

All global shortcuts use Ctrl+Alt+ modifier. Implemented via low-level keyboard hook (not glfwSetKeyCallback) so they work even when Tubelight has no focus.

Control: - Ctrl+Alt+Q — quit - Ctrl+Alt+M — toggle in-app menu - Ctrl+Alt+F — toggle freeze frame (debug-friendly, no autorepeat flipping) - Ctrl+Alt+Enter — toggle runtime fullscreen preserving aspect - Ctrl+Alt+T — attach/detach foreground window (target mode) - Ctrl+Alt+H — toggle status HUD (top-right)

Capture: - Ctrl+Alt+S — PNG screenshot (async encode) - Ctrl+Alt+V — toggle MP4 recording (needs ffmpeg)

Pipeline: - Ctrl+Alt+0 — re-enable all passes - Ctrl+Alt+1 .. Ctrl+Alt+8 — toggle individual pass (-1 signal .. 6 composition)

Removed in v0.1.4 (ADR-0001 — behaviour is now always-on by design): - ~~Ctrl+Alt+C~~ click-through: now mode-dependent (windowed=off; target/region/fullscreen=on). - ~~Ctrl+Alt+R~~ recordable mode: always on. Snipping Tool / Game Bar / OBS see the overlay automatically.

In --shader-only mode (debug): direct keys without modifier. - ESC quit - 1..8 toggle pass - 0 enable all - R reload shaders

12. Typical use cases

Recipe-book by content type. Each combination includes recommended profile + signal, suggested aspect ratio, and optional notes.

12.1 Standalone emulators

Mednafen (PCE / NES / SNES / SMS / GG / Saturn / PSX): profile pvm-8220 + signal composite_ntsc for US market, commodore-1084s + composite_pal for PAL. Aspect: 4:3. Mode: --overlay-target Mednafen or --overlay-target-pid <PID>.

MAME (vertical or horizontal arcade): profile wells-gardner-k7000 + signal composite_ntsc (US arcades) or rgb_vga (PC CRTs for Neo Geo on MVS). Aspect: game-dependent (3:4 vertical, 4:3 horizontal). Mode: target or region.

openMSX: profile commodore-1084s (consumer monitors closest to European MSX setup) + signal composite_pal. Aspect: 4:3.

BlastEm (Mega Drive): profile pvm-20m4 (broadcast 20") + signal composite_ntsc. Mega Drive output composite by default; some sets could output SCART RGB on PAL. Aspect: 4:3.

FCEUX / Nestopia (NES): profile pvm-8220 + signal composite_ntsc. NES output very noisy composite; for full authenticity, use rf (what 1985 kids saw with the TV connected via antenna).

Snes9x (SNES): profile pvm-8220 + signal svideo. SNES output native S-Video with the right cable. Aspect: 4:3 (some PAL games are 5:4).

12.2 RetroArch / libretro

Option A — overlay on top of RetroArch: launch RetroArch without shaders, launch Tubelight in target mode on top. Works but loses integration with RetroArch's stateful shader pipeline.

Option B — export .slangp preset from Tubelight to load in RetroArch:

tubelight --export-slangp my-pvm.slangp --profile pvm-8220 --signal composite_ntsc

Copy the .slangp and generated .slang shaders to RetroArch's shaders/ folder and load the preset from Settings → Video → Shaders → Load Preset. Less faithful than the overlay (RetroArch runs the shaders in its own pipeline, not Tubelight's) but integrates with savestate, threaded video, etc.

12.3 Modern retro games

Pixel art or intentionally "retro" low-poly — UFO 50, Sea of Stars, Shovel Knight, Crypt of the NecroDancer, Stardew Valley, Eastward, Hyper Light Drifter, Celeste, Children of Morta.

PVM-8220 + composite NTSC for US console feel. Commodore 1084S + composite PAL for European home computer feel. Wells-Gardner K7000 + rgb_vga for arcade feel. Terminal P31 + rgb_vga for an absurd CRT-monochrome vibe over your modern pixel art (not the original medium but it looks good).

Aspect: the game's (most are 16:9 native; pixel-art ones are 16:9 with bars or 4:3 scaled).

12.4 Video players

VHS / 70s-80s TV archive material: profile tv-bw-p4 or commodore-1084s + signal rf. Brutal on old demos, retro newscasts, ads.

Vintage music videos / broadcast clips: profile pvm-20m4 or bvm-20f1u + signal composite_ntsc. For broadcast archive material.

YouTube / modern streaming: any profile + signal rgb_vga or component. Tubelight won't degrade the image.

12.5 Terminals and code

PowerShell / cmd / WSL: profile terminal-p31 (green) or terminal-p3-amber (amber) + signal rgb_vga. Posterize 6 default. Persistence 0.1-0.3 so the blinking cursor leaves minimal halo.

VS Code / editors: same. For a "1985 hacker" feel without giving up modern extensions, terminal-p31 profile with high bloom and persistence 0.3-0.4 gives the Matrix look without caricature.

Remote server terminal via SSH: same as local. Tubelight neither knows nor cares that it's remote.

13. Troubleshooting

Most common problems and how to fix them. If yours isn't here, open a GitHub issue.

13.1 Overlay black / not updating

Symptom: Tubelight window shows black or frozen on first frame.

Typical cause 1: recordable mode previously enabled + WDA dropped + DXGI loop. v0.1.0 forces recordable OFF at startup to fix this. On older builds, edit %APPDATA%\Tubelight\settings.json and set "recordable": false.

Cause 2: another overlay on top (Discord overlay, Nvidia Shadowplay, Steam overlay). DWM composes first and Tubelight sees the composite of all. Disable those overlays.

Cause 3: outdated GL driver. Tubelight requires OpenGL 4.5 core. Update the GPU driver.

13.2 Perceived lag in what's below

Symptom: the game/terminal underneath runs one frame behind.

Cause: Tubelight adds ~1 DXGI frame + ~1 composite frame. At 60 Hz = ~33 ms. Expected.

Mitigation: enable low_latency in the Image tab (vsync OFF + 240 FPS cap). Latency drops to ~4-8 ms at the cost of higher GPU use. Warning: setting low_latency=true and then manually disabling the cap will spin the main loop at thousands of FPS, saturating CPU+GPU. v0.1.0 forces a 240 cap if low_latency is ON.

13.3 "ffmpeg not found" when recording

Symptom: Ctrl+Alt+V shows toast ffmpeg not found in PATH.

Fix: install ffmpeg via winget (winget install Gyan.FFmpeg), chocolatey, or manual download from gyan.dev. Restart Tubelight after install (PATH is cached per process).

PS> winget install Gyan.FFmpeg

13.4 Profile won't load

Symptom: CRT dropdown loads a profile and nothing changes / toast with validation error.

Fix: validate the JSON with tubelight --validate-profile profiles/crts/<id>.json. Tells you exactly which field is wrong or fails schema. Compare against a bundled profile to see what a valid JSON looks like.

> tubelight --validate-profile profiles/crts/pvm-8220.json

13.5 STATUS_STACK_BUFFER_OVERRUN on recordable toggle (legacy)

Symptom: in pre-v0.1.0 builds, toggling recordable mode crashed with /GS stack-canary fail (STATUS_STACK_BUFFER_OVERRUN).

Fix: v0.1.0 final fixed it by moving MagCapture to heap-allocated unique_ptr (was stack-allocated near the canary), calling cpu_buffer_.resize() BEFORE MagSetImageScalingCallback, adding destructor that calls shutdown(), and making filter_hwnds_[1] a persistent member. Upgrade if you're on an old build.

13.6 Win+Shift+S doesn't show overlay — checklist

  1. Did you toggle Ctrl+Alt+R (recordable mode)? Without it, Snipping Tool sees the app underneath, not the overlay.
  2. Did the toast confirm "Recordable mode ON"?
  3. Is Tubelight actually on top of the area you're capturing? Move the window or use overlay-fullscreen.
  4. If all OK and the raw content still shows: close Snipping Tool, wait 2 s, try again (the app caches state).

13.7 Game Bar doesn't record overlay — checklist

Same problem and solution as §13.6 — Game Bar captures via DXGI Desktop Duplication or Windows.Graphics.Capture; both respect WDA_EXCLUDEFROMCAPTURE.

Fix: toggle Ctrl+Alt+R before starting Game Bar recording (Win+G → Capture → Record).

13.8 OBS does record (Window Capture WGC) — instructions

14. Glossary

Aperture grille

CRT mask structure made of fine vertical wires (Trinitron). Yields bright color and crisp verticals at the cost of softer horizontals and two thin damping wires visible against bright fields.

Shadow mask

Perforated plate behind the phosphor in conventional CRTs. Each R-G-B triad shares one hole; the pattern reads as a dot grid in bright areas.

Slot mask

Half-way between shadow mask and aperture grille: vertical slots instead of dots or wires. Typical of 80s–90s TVs and consumer monitors (Commodore 1084, many consumer PVMs).

P22

Standard color-TV phosphor in Europe and the US. Three components (R, G, B) with distinct decay times: red persists longer than green/blue, producing the characteristic reddish trail behind fast motion.

P31

Standard green phosphor (Apple //e, VT100, oscilloscopes). Medium persistence, peak at 525 nm. Highly saturated towards pure green.

P3

Amber phosphor used in IBM 5151 and WYSE terminals. Warm chromaticity (high R, mid G), reduces eye fatigue over long sessions per 1980s ergonomic guidance.

P4

White phosphor of black-and-white TV. Balance close to natural blueish white. Found in B/W TVs and the Macintosh Classic.

P1

Oscilloscope green phosphor, long persistence. A fast trace leaves a visible afterglow for many milliseconds.

Persistence

Time during which the phosphor keeps emitting light after the beam passes. Determines motion trail. Per-channel: P22's slow R yields the typical reddish trail.

Scanline

Each of the horizontal lines drawn by the CRT beam per frame. NTSC: 240 visible, PAL: 288, VGA: 480.

Dot pitch

Distance between phosphor triads on the mask (mm). Smaller = higher effective resolution. PVM-8220: 0.30 mm; BVM-20F1U: 0.31 mm Super Fine Pitch; NEC 4FG: 0.28 mm.

CRT gamma

Non-linear curve of the CRT gun (≈2.2–2.5). Maps beam voltage to luminance. Distinct from modern display gamma (sRGB ≈ 2.2). Tubelight models it separately from output correction.

Trinitron

Sony aperture-grille tech, dominant in pro and gaming monitors since the 80s. Visible cue: bright image, crisp verticals, two damping wires crossing horizontally.

Halation

Diffuse halo around bright areas caused by glass internal reflection and phosphor scattering. Distinct from bloom: halation skews red because R travels farther.

Bloom

Luminance bleed in saturated regions: bright pixels appear to grow. In real CRTs caused by beam spread; in Tubelight modeled as threshold-gated Gaussian blur.

Barrel distortion

Geometric image curvature, more pronounced on old TVs and arcades. Tubelight applies it as configurable mild radial distortion.

Vignette

Brightness falloff towards the corners. In physical CRTs caused by tube geometry and bezel. Subtle on PVMs, strong on arcades.

Bezel

Physical CRT frame. Tubelight v0.1.0 omits it by default (overlay aims to be the glass, not the furniture). PNG bezel planned for v1.1.

WDA_EXCLUDEFROMCAPTURE

DWM flag (Win10 build 2004+) that excludes a window from ALL desktop-level capturers (Snipping Tool, Game Bar, OBS Display Capture). Binary, not per-capturer. Tubelight uses it by default to prevent a feedback loop with the captured area.

Magnification API

Pre-DWM Win32 accessibility API; the only documented way to exclude one specific window from its own capturer (not from the rest of the system). Tubelight's recordable mode uses it.

DXGI Desktop Duplication

DirectX API to capture the DWM desktop composite with very low latency (~1 frame). Tubelight uses it to extract the content to feed into the pipeline. Works with any app (DX/GL/Vulkan) because it captures DWM's final composite.

Posterize

Reduction of tonal levels. Tubelight uses it to emulate 1-bit displays (Mac Classic = 2 levels), terminals (6) and P1 oscilloscope phosphor. 0 = analog (no posterization).

Voltage bloom

Dynamic beam-width increase when the frame mean is very bright: the gun overloads, the beam widens, the image softens. Real in physical CRTs; planned as optional v1.1 effect.

SMPTE-C primaries

xy chromaticity of R/G/B phosphors defined by SMPTE for pro NTSC 525-line. Distinct from Rec.601, EBU PAL, and from the actual phosphors of each CRT.

DWM

Desktop Window Manager: Windows' desktop compositor since Vista. Renders each window to its own buffer and composes the result with effects. Every desktop-level capturer (except Magnification API) goes through DWM, with implications for recordable mode.
